# 什么阶段开始值得接入 Monitor Lite

## 核心判断
`openclaw-monitor-lite` 并不是给刚看 OpenClaw 教程的人准备的，
而是给已经进入真实运行阶段的人准备的。

---

## 开始值得接入的 4 个信号
### 1. 你已经开始跑 cron / session / tool 调用
这时候就会出现：
- 哪些任务失败了
- 哪些任务在持续花钱
- 最近到底发生了什么

### 2. 你开始关心成本
一旦开始反复运行模型调用，成本不再是抽象数字，而是需要持续观察的东西。

### 3. 你开始关心异常提醒
如果任务失败只能靠手翻日志，说明已经进入需要告警的阶段。

### 4. 你开始有多个节点 / 多个运行面
一旦运行复杂度上升，没有统一视图就会明显增加排障成本。

---

## 暂时不急着接入的情况
- 还停留在围观和教程阶段
- 只在单机做极轻量试玩
- 还没有真实运行成本、告警和健康度问题

---

## 结论
这个项目最适合的时机不是“刚听说 OpenClaw 的时候”，而是：

> **当你已经开始真正运行 OpenClaw，并且开始想知道它到底跑得怎么样。**
