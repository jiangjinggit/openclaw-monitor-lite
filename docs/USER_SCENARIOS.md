# OpenClaw Monitor Lite 样例用户 / 使用场景

## 场景 1：独立开发者开始持续跑 cron
- 已经在真实运行 OpenClaw
- 需要知道哪些任务失败、哪些任务在持续花钱
- 适合先用 Monitor Lite 看运行全局

## 场景 2：小团队开始依赖多个节点和渠道
- 已接入多节点、多消息渠道或多个运行面
- 需要更快发现异常和节点不稳定问题
- 适合用它做第一层运营控制台

## 场景 3：给客户交付后需要一个可见后台
- 不希望交付后只能靠终端日志排查
- 需要一个更像运营后台的可见视图
- 适合拿它承接持续运行阶段

## 场景 4：已经开始关心成本和异常趋势
- 不只是想看“现在有没有挂”
- 还想看成本变化、失败趋势和告警提醒
- 这就是 Agent Ops Lite 最适合切入的阶段
