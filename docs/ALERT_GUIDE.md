# 告警功能使用指南

## 概述

OpenClaw Monitor Lite 现在支持告警和通知功能，可以在出现问题时及时通知你。

## 功能特性

### 支持的告警类型

1. **成本超限告警**
   - 监控日成本是否超过设定阈值
   - 默认阈值：¥100/天

2. **错误率告警**
   - 监控任务失败率
   - 默认阈值：20%

3. **Cron 连续失败告警**
   - 监控 Cron 任务连续失败次数
   - 默认阈值：连续失败 3 次

4. **Session 超时告警**（可选）
   - 监控 Session 运行时间
   - 默认阈值：1 小时

### 支持的通知渠道

1. **Telegram Bot**
   - 通过 Telegram Bot 发送通知
   - 需要配置 Bot Token 和 Chat ID

2. **飞书 Webhook**
   - 通过飞书机器人发送通知
   - 需要配置 Webhook URL

3. **邮件通知**（计划中）
4. **自定义 Webhook**（计划中）

## 快速开始

### 1. 访问告警配置页面

启动服务后，访问：
```
http://localhost:4311/alerts.html
```

或者在首页点击"告警配置"按钮。

### 2. 配置 Telegram 通知

#### 创建 Telegram Bot

1. 在 Telegram 中搜索 `@BotFather`
2. 发送 `/newbot` 创建新机器人
3. 按提示设置机器人名称
4. 获取 Bot Token（格式：`123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`）

#### 获取 Chat ID

1. 在 Telegram 中搜索 `@userinfobot`
2. 发送任意消息
3. 获取你的 Chat ID（格式：`123456789`）

#### 配置

1. 在告警配置页面找到"Telegram"部分
2. 勾选"启用 Telegram 通知"
3. 填入 Bot Token 和 Chat ID
4. 点击"测试通知"验证配置
5. 点击"保存配置"

### 3. 配置飞书通知

#### 创建飞书机器人

1. 在飞书群聊中，点击右上角设置
2. 选择"群机器人" → "添加机器人"
3. 选择"自定义机器人"
4. 设置机器人名称和描述
5. 复制 Webhook URL

#### 配置

1. 在告警配置页面找到"飞书"部分
2. 勾选"启用飞书通知"
3. 填入 Webhook URL
4. 点击"测试通知"验证配置
5. 点击"保存配置"

### 4. 管理告警规则

#### 启用/禁用规则

在"告警规则"部分，每个规则右侧有一个开关：
- 绿色：已启用
- 灰色：已禁用

点击开关可以切换状态。

#### 规则说明

每个规则卡片显示：
- **规则名称**：告警的名称
- **描述**：告警的详细说明
- **类型**：告警类型（cost/error_rate/cron_failure 等）
- **条件**：触发条件（如 daily_cost > 100）
- **严重程度**：critical（严重）/ error（错误）/ warning（警告）
- **通知渠道**：使用哪些渠道发送通知
- **冷却时间**：两次告警之间的最小间隔（秒）

### 5. 查看告警历史

在"告警历史"部分可以看到：
- 触发时间
- 规则名称
- 严重程度
- 当前值
- 阈值

历史记录按时间倒序排列，最多显示最近 50 条。

## 告警检查机制

### 自动检查

告警检查脚本会在以下时机自动运行：
1. 每次数据同步后
2. 定期检查（可配置间隔）

### 手动检查

在告警配置页面点击"立即检查"按钮，可以手动触发一次告警检查。

### 冷却机制

每个规则都有冷却时间（cooldown），在冷却期内不会重复发送相同的告警。

例如：
- 成本超限告警：冷却 1 小时（3600 秒）
- 错误率告警：冷却 30 分钟（1800 秒）
- Cron 失败告警：冷却 10 分钟（600 秒）

## 自定义告警规则

目前告警规则通过配置文件管理，位于：
```
data/alert-rules.json
```

### 规则结构

```json
{
  "id": "rule-id",
  "name": "规则名称",
  "description": "规则描述",
  "type": "告警类型",
  "condition": {
    "metric": "指标名称",
    "operator": "比较运算符",
    "threshold": 阈值
  },
  "enabled": true,
  "channels": ["telegram", "feishu"],
  "severity": "warning",
  "cooldown": 3600
}
```

### 支持的运算符

- `>` : 大于
- `>=` : 大于等于
- `<` : 小于
- `<=` : 小于等于
- `==` : 等于
- `!=` : 不等于

### 添加自定义规则

1. 编辑 `data/alert-rules.json`
2. 在 `rules` 数组中添加新规则
3. 重启服务或刷新页面

## 故障排查

### Telegram 通知发送失败

1. 检查 Bot Token 是否正确
2. 检查 Chat ID 是否正确
3. 确保机器人已经和你的账号建立对话（发送 `/start` 给机器人）
4. 检查网络连接

### 飞书通知发送失败

1. 检查 Webhook URL 是否正确
2. 确保 Webhook 未过期
3. 检查网络连接

### 告警未触发

1. 检查规则是否已启用
2. 检查是否在冷却期内
3. 检查条件是否满足
4. 手动运行 `node scripts/check-alerts.js` 查看日志

## 最佳实践

### 1. 合理设置阈值

- 成本阈值：根据实际预算设置
- 错误率阈值：建议 10%-20%
- Cron 失败次数：建议 2-3 次

### 2. 选择合适的通知渠道

- 紧急告警：Telegram（实时推送）
- 团队协作：飞书（群聊通知）
- 记录存档：邮件（计划中）

### 3. 避免告警疲劳

- 设置合理的冷却时间
- 只启用真正重要的规则
- 定期检查告警历史，调整阈值

### 4. 定期测试

- 每周测试一次通知渠道
- 确保配置正确
- 验证告警能够正常触发

## 技术细节

### 文件结构

```
data/
  alert-rules.json          # 告警规则配置
  alert-history.json        # 告警历史记录
  notification-config.json  # 通知渠道配置

scripts/
  check-alerts.js          # 告警检查脚本

public/
  alerts.html              # 告警配置页面
```

### API 端点

- `GET /api/alert-rules` - 获取告警规则
- `POST /api/alert-rules` - 更新告警规则
- `GET /api/alert-history` - 获取告警历史
- `GET /api/notification-config` - 获取通知配置
- `POST /api/notification-config` - 更新通知配置
- `POST /api/check-alerts` - 手动触发告警检查
- `POST /api/test-notification` - 测试通知
