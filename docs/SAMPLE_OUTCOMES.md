# OpenClaw Monitor Lite 样例结果

## 用户接入后，最希望更快看到什么
### 1. 今天花了多少钱
不是只知道系统在跑，而是知道：
- 今天成本大概多少
- 哪些任务值得继续观察

### 2. 哪些任务在失败
不是等用户反馈后才知道，而是更早看到：
- 最近失败任务
- 失败频率
- 哪些任务最值得优先排查

### 3. 哪些节点不稳定
不是故障后再翻日志，而是提前看到：
- 哪些节点波动更大
- 哪些运行面更容易出问题

### 4. 最近到底出了什么错
不是零散看终端，而是统一看到：
- 错误摘要
- 异常趋势
- 更适合后续持续排障

---

## 结果价值
`monitor-lite` 更像把“持续运行中的不确定性”收成一个更容易看的运营视图。
