# OpenClaw Agent Ops Lite 定位

## 核心定位
`openclaw-monitor-lite` 当前最适合承担的角色不是短期首单，而是：

> **OpenClaw 的长期主壳 / Agent Ops Lite / 轻量运营控制台**

---

## 它主要解决什么问题
当 OpenClaw 从“能跑”进入“持续运行”阶段，用户最常遇到的不是功能不够，而是：
- 成本看不清
- 异常没人提醒
- 节点状态看不见
- 错误日志难排查

这个项目就是用更轻、更贴 OpenClaw 场景的方式，补这层运行视图。

---

## 当前最该优先卖的 4 个点
1. 成本可见
2. 告警可达
3. 节点健康可见
4. 错误日志可追

---

## 和产品矩阵的关系
- `openclaw-security-audit-lite`：更适合短期首单 / 体检 / 基线 / 风险报告
- `openclaw-monitor-lite`：更适合长期主壳 / 订阅 / 持续运营层
- `openclaw-template-market`：更适合模板前端和增长入口

---

## 当前策略
先把首页和产品表达收口成：
- 轻量运营控制台
- Agent Ops Lite
- 成本 / 告警 / 节点健康 / 错误日志

先卖清楚“看得见、查得到、能提醒”，再决定扩更复杂的控制层。
