# OpenClaw Monitor Lite 首版升级路径

## 当前最合理的升级顺序
### 1. 单人运行视图
适合独立开发者：
- 看成本
- 看告警
- 看节点健康
- 看错误日志

### 2. 多节点 / 多任务视图
适合已经开始同时跑多个节点、cron、session 的用户：
- 看多节点状态
- 看异常聚合
- 看更明显的运行复杂度变化

### 3. 团队版运行视图
适合小团队和交付场景：
- 更适合共享查看
- 更适合阶段性汇报
- 更适合长期运行观察

### 4. 长期 Agent Ops 主壳
适合作为持续运行层：
- 周期性查看
- 更长期留存
- 后续可继续扩更多能力

---

## 设计原则
先卖最轻的运行视图，
再逐步升级到更多节点、更多团队、更多长期能力。
