为什么它是长期主壳，不是短期首单

短期首单`security-audit-lite` 更适合卖体检、风险评分和修复建议，首次购买更容易发生。
长期主壳`monitor-lite` 更适合承接持续运行阶段：成本、告警、节点健康和错误日志。
先后顺序先打首单，再承接长期运行层，这样分工更清楚、也更符合真实购买路径。

什么阶段开始值得接入

开始跑 cron / session一旦已经进入真实运行阶段，就会开始关心哪些任务失败、哪些任务在持续花钱。
开始关心成本和异常如果已经需要知道今天花了多少、哪些异常需要提醒，就说明值得接入。
开始有多个节点或运行面一旦复杂度上升，没有统一视图就会明显增加排障成本。

为什么它比通用 observability 工具更适合 OpenClaw

更贴运行对象直接围绕 cron、session、node 和 runtime 状态组织，而不是先讲抽象 trace。
更轻先卖成本、告警、节点健康、错误日志四个最现实的运行视图。
更适合作为主壳和安全体检模块分工明确，承担持续运行层，而不是首单体检层。
